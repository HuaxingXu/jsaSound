jsaSound for Interactive Sound Modeling

jsaSound assists interactive sound designers in providing sound users (game, web, application developers) with models that have a simple and consistent API exposing exactly the interaction the model designer intends.

The Sound Model API:
play()
release()
stop()

setParam([name, number],val)
setParamNorm([name, number], val)

getParam([name, number], ["name", "type", "val", "normval", "min","max"]) // type can return "range" or "url"

getNumParams()
getAboutText();

You might notice that the sound model for this demo is "served from the cloud" - your apps can use the sound models this way, too!

jsaSound also supports sound model developers with "composing" sound models in chains and heirarchies (using Kumar's very cool "GraphNode" for wrapping models to make them look like Audio Nodes), and for scheduling events in a varity of ways (see the rhythmic pattern models).
